# 题库重构与实现说明

## 一、对 brainstorm 文档的核心理解

题库不应只按主题词汇或语法点组织，而应优先回答一个问题：

> 学习者在真实交流或写作中，需要完成什么交际任务？

因此，新版题库以十大主要表达类型为主轴：

1. 信息陈述与客观描述
2. 行为、过程与事件表达
3. 叙事、经历与个人背景
4. 知识说明与概念解释
5. 个人状态、需求、偏好与计划
6. 观点、评价与完整论证
7. 问题分析、解决方案与决策
8. 预测、假设与可能性表达
9. 人际互动与事务沟通
10. 语篇组织、表达调整与交流修复

主题只负责提供内容，主要表达类型负责决定训练目标。

## 二、旧题库的主要不足

1. 过度集中于议论文模板。
2. 口语、经历叙述、日常事务和交流修复不足。
3. 部分选项是过长的半句或从句，降低了语序训练价值。
4. 干扰项使用独立 kind 时，可能通过颜色暴露答案。
5. 主题与表达功能之间缺少明确标签。
6. 每个主题机械套用相同写作结构，真实场景不足。

## 三、新版实现

### 1. 题目规模

- 12 个互补主题；
- 每个主题 55 道拼句题；
- 拼句题共 660 道；
- 每个主题 12 道自由表达题；
- 自由表达题共 144 道；
- 总题数 804 道。

### 2. 拼句题颗粒度

- 所有 token 为单词或短语；
- 每个 token 最多约 5 个英文单词；
- 标点附着在相邻短语中，不单独作为选项；
- 长从句拆成多个结构或内容语块；
- 每题加入少量自然干扰项。

### 3. 颜色规则

token.kind 只能是：

- `structure`：结构、连接和功能语块；
- `concept`：主题内容、动作、对象和概念。

干扰项也使用这两种 kind，不存在 `distractor` 类别，不获得独立颜色。

### 4. 中文题干形式

- 中文直接表达；
- 问题回答；
- 情景任务；
- 上下文衔接。

### 5. 次级标签

每题包含：

- 逻辑关系；
- 表达来源；
- 使用场景；
- 语体；
- 语气；
- 时间与确定性；
- 表达长度。

### 6. 口语与写作

题目通过 mode 标记：

- `speaking`
- `writing`
- `both`

口语题包括个人回答、经历、请求、确认、安排、投诉、不同意、折中、澄清和总结。

写作题包括客观描述、数据趋势、概念定义、完整论证、问题解决、预测和有限定结论。

## 四、页面修改

1. 练习页增加表达类型和口语/写作模式筛选。
2. 题目头部只显示必要标签。
3. 拼句选项只显示结构色和概念色。
4. 题目选项顺序在会话中打乱。
5. 页面提示使用组件和 toast，不使用浏览器 alert。
6. 自由表达题保持原有保存、插入和 AI 评判功能。
