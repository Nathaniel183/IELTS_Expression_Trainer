# 数据格式说明

## questions.txt

### build 题核心字段

```json
{
  "id": "education-build-001",
  "themeId": "education",
  "type": "build",
  "promptZh": "中文题干",
  "tokens": [
    {"id":"c1","text":"Online learning","kind":"concept"},
    {"id":"c2","text":"refers to","kind":"structure"}
  ],
  "acceptableAnswers": [["c1","c2"]],
  "sampleAnswers": ["Online learning refers to ..."],
  "cloze": {
    "parts": [],
    "items": []
  }
}
```

`kind` 只能使用：

- `structure`
- `concept`

干扰项也必须使用这两个类别，不能建立专属干扰项颜色。

### cloze.parts

- `text`：直接显示的句子内容；
- `blank`：显示输入框，通过 id 引用 cloze.items。

### cloze.items

- `answer`：标准答案；
- `accepted`：可接受答案数组；
- `zh`：中文提示；
- `kind`：structure 或 concept；
- `wordCount`：词数，用于输入框宽度和提示。

## fallback-data.js

修改任何 data 文件后，应同步更新 fallback-data.js，保证直接双击 HTML 时也能使用最新数据。

## 数据检查

运行：

```bash
python tools/validate_data.py
```
