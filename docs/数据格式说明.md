# 数据格式说明

## questions.txt 的 build 题

```json
{
  "id": "education-build-001",
  "themeId": "education",
  "type": "build",
  "primaryFunction": "objective-description",
  "subFunction": "功能描述",
  "mode": "writing",
  "promptForm": "中文直接表达",
  "promptZh": "请根据中文拼出自然英文：……",
  "secondaryTags": {
    "logic": "整体部分",
    "register": "一般书面语"
  },
  "tokens": [
    {"id": "t01", "text": "Online learning", "kind": "concept"},
    {"id": "t02", "text": "is mainly used to", "kind": "structure"}
  ],
  "acceptableAnswers": [["t01", "t02"]],
  "sampleAnswers": ["Online learning is mainly used to ..."],
  "explanation": "训练事物功能描述。"
}
```

`kind` 只能使用：

- `structure`
- `concept`

不要使用 `distractor` 作为 kind。干扰项和答案项使用相同类别，页面不会给干扰项特殊颜色。

每个 token 建议不超过约五个英文单词。完整长从句应拆成多个短语。

## free 题

自由表达题保留：

- `suggestedTemplateIds`
- `suggestedVocabIds`
- `sampleAnswer`
- `rubric`

## 校验

运行：

```bash
python tools/validate_data.py
```

校验内容包括：

- JSON 格式；
- ID 唯一性；
- 主题、模板和词汇引用；
- 拼句答案 token；
- token kind；
- token 长度。
